# **1\. Core Scoring Model**

All phases use **three possible backend scores only**:

* **1** \= Skill demonstrated effectively

* **0.5** \= Skill partially demonstrated, delayed, incomplete, or out of order

* **0** \= Skill not demonstrated

No other numeric values are permitted in the 1st Skills Practice.

---

# **2\. Binary Feedback Mapping Rule**

All formative feedback must follow this mapping:

| Backend Score | Feedback Type |
| ----- | ----- |
| 1 | Keep Doing only |
| 0.5 | Keep Doing \+ Try Next Time |
| 0 | Try Next Time |

Important clarifications:

* “Keep Doing” always appears first.

* If score \= 0, do **not** fabricate strengths.

* Numeric scores are never referenced in user-facing feedback.

---

# **3\. Phase-Level Evaluation Logic**

## **3.1 Phase 1 – Counsel**

### **Evaluate for 5 Required Elements:**

* Child’s age

* Cancer prevention

* Vaccine safety

* Recommendation today

* Return for second dose

### **Scoring**

* 1 \= At least 4–5 elements included correctly

* 0.5 \= 1–3 elements included or partially correct

* 0 \= No structured Counsel statement

### **Feedback**

* 1 → Keep Doing only

* 0.5 → Keep Doing \+ Try Next Time

* 0 → Try Next Time

---

## **3.2 Phase 2 – Listen**

### **Evaluate:**

Was an active Listen skill used before answering?

* (Explore)

* (Restate)

### **Scoring**

* 1 \= Appropriate Listen skill used before answering

* 0.5 \= Listen skill used but incomplete, interrupted, or poorly timed

* 0 \= No Listen skill used

### **Feedback**

* 1 → Keep Doing only

* 0.5 → Keep Doing \+ Try Next Time

* 0 → Try Next Time

---

## **3.3 Phase 3 – Empathize**

### **Evaluate:**

Was an Empathize skill used before answering?

* (Acknowledge)

* (Normalize)

* (Validate)

Order and timing affect backend scoring but are not explained to user.

### 

### **Scoring**

* 1 \= Empathize skill used appropriately before answering

* 0.5 \= Empathize skill used but delayed, incomplete, or out of order

* 0 \= No Empathize skill used

### **Feedback**

* 1 → Keep Doing only

* 0.5 → Keep Doing \+ Try Next Time

* 0 → Try Next Time

Frontend feedback must not reference:

* “Out of order”

* “Second chance”

* Timing distinctions

---

## **3.4 Phase 4 – Answer \+ Recommend**

### **Evaluate two required components:**

1. Clear, accurate answer

2. Strong recommendation (“I recommend…” or equivalent)

### **Scoring**

Phase receives one combined score:

* 1 \= Accurate answer \+ strong recommendation

* 0.5 \= One component missing, weak, or delayed

* 0 \= Neither component adequately performed

If inaccurate medical content is detected:

* Score \= 0.5 or 0 depending on severity

* Flag for research log

* Provide corrective Try Next Time guidance

* Do not use language like “incorrect” or “wrong”

### **Feedback**

* 1 → Keep Doing only

* 0.5 → Keep Doing \+ Try Next Time

* 0 → Try Next Time

---

# **4\. Formative Feedback Structure Rules**

All formative feedback must follow this structure:

1. Keep Doing statement (if score ≥ 0.5)

2. Try Next Time statement (if score ≤ 0.5)

If score \= 0:

* Do not fabricate strengths.

* Provide clear Try Next Time guidance.

Feedback must:

* Be plain language

* Be behavior-specific

* Be supportive

* Avoid judgmental phrasing

* Avoid grading language

Prohibited:

* Numeric scores

* Fractions

* “Partial credit”

* “Incorrect”

* “Out of order”

* “Second attempt”

* Any grading terminology

---

# 

# **5\. Summative Feedback (End of Session)**

Structure:

1. Affirm participation in practice

2. Identify 2–3 key skills to continue strengthening

3. Provide forward-looking guidance for 2nd Skills Practice

Rules:

* No numeric totals

* No performance summaries

* No phase-by-phase recap

Focus on:

* Patterns

* Skill development

* Readiness for independent practice

---

# **6\. Data Separation Rule**

Two separate system layers must exist:

### **Backend**

* Numeric scores

* Skill detection markers

* Flags

* Timestamps

### **Frontend**

* Qualitative coaching statements only

* No visibility of backend scoring

---

# **7\. Non-Inheritance Clause**

This 1 / 0.5 / 0 model applies only to:

* 1st Skills Practice (Training)

The 2nd Skills Practice (Booster):

* Will not use phase-level formative feedback

* Will not expose phase-level structure

* Will use separate summative-only logic
